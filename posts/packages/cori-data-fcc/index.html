Overview

The cori.data.fcc package provides tools for downloading, processing, and analyzing FCC broadband data in R. It simplifies access to the National Broadband Map and other FCC data sources, making it easier to conduct broadband equity and availability analysis.

NoteWhy Use This Package
  • Simplified data access - Download FCC data directly from R
  • Standardized processing - Consistent data cleaning and formatting
  • Analysis tools - Built-in functions for coverage gaps and equity analysis
  • Integration ready - Works seamlessly with CORI data pipelines

Quick Start

library(cori.data.fcc)

# Download broadband availability data for a state
data <- get_broadband_data(state = "VT")

# Analyze coverage by technology type
coverage_summary <- summarize_coverage(data, by = "technology")

# Create coverage map
plot_coverage(data, metric = "download_speed")

Key Functions

Data Access

# Download broadband availability data
get_broadband_data(
  state = "VT",           # Two-letter state code
  geography = "tract",    # "tract", "block", or "county"
  vintage = "latest"      # FCC data version
)

Analysis Functions

# Aggregate coverage statistics
summarize_coverage(
  data,
  by = "technology"      # "technology", "provider", or "speed_tier"
)

# Identify coverage gaps
calculate_gaps(
  data,
  threshold = 25/3       # Download/upload speed threshold in Mbps
)

Visualization

# Create coverage visualizations
plot_coverage(
  data,
  metric = "download_speed",    # Metric to visualize
  geography = "tract"            # Geographic level
)

Common Use Cases

State-Level Coverage Analysis

library(dplyr)

# Get Vermont broadband data
vt_data <- get_broadband_data(state = "VT")

# Calculate coverage rates
coverage_rates <- vt_data |>
  summarize_coverage(by = "technology") |>
  arrange(desc(coverage_pct))

Identifying Coverage Gaps

# Find areas lacking 25/3 Mbps service
gaps_25_3 <- calculate_gaps(
  vt_data,
  threshold = list(download = 25, upload = 3)
)

# Find areas lacking 100/20 Mbps service
gaps_100_20 <- calculate_gaps(
  vt_data,
  threshold = list(download = 100, upload = 20)
)

Technology Comparison

# Compare fiber vs. cable availability
tech_comparison <- vt_data |>
  filter(technology %in% c("Fiber", "Cable")) |>
  group_by(technology) |>
  summarize(
    avg_download = mean(download_speed, na.rm = TRUE),
    avg_upload = mean(upload_speed, na.rm = TRUE),
    coverage_pct = sum(covered) / n() * 100
  )

Data Structure

Standard Output Format:

  • One row per geographic unit per provider per technology
  • Standardized variable names
  • Geographic identifiers (GEOID)
  • Speed tiers and technology types

Typical Variables:

geoid, state, county, tract, block, provider_name, technology,
download_speed, upload_speed, covered, available_speeds

Integration with CORI Workflows

With Rural Definitions

library(ruraldefinitions)

# Get broadband data
broadband <- get_broadband_data(state = "VT")

# Add rural classification
broadband_rural <- broadband |>
  left_join(
    get_definition("ruca", 2020),
    by = "geoid"
  )

# Compare rural vs. urban coverage
coverage_by_rural <- broadband_rural |>
  group_by(is_rural) |>
  summarize_coverage()

With Census Data

library(tidycensus)

# Get population data
pop_data <- get_acs(
  geography = "tract",
  variables = "B01001_001",
  state = "VT"
)

# Join with broadband data
broadband_pop <- broadband |>
  left_join(pop_data, by = "geoid") |>
  mutate(
    unserved_population = population * (1 - coverage_pct / 100)
  )

Data Sources

The package accesses:

  • FCC National Broadband Map - Broadband availability and speed data
  • FCC Form 477 - Provider coverage submissions (historical)
  • BDC (Broadband Data Collection) - Updated availability data (current)

Performance Considerations

TipOptimization Tips
  1. Download by state - Reduces data transfer and processing time
  2. Use appropriate geography - County is faster than block for large-scale analysis
  3. Filter early - Subset data before complex calculations
  4. Cache results - Save processed data locally for repeated analysis

Known Limitations

WarningImportant Notes
  • Coverage vs. Adoption - Data shows availability, not actual subscriptions
  • Provider reporting - Based on provider submissions, may contain inaccuracies
  • Update lag - Data may be 6-12 months behind current conditions
  • Block-level challenges - Census blocks can be very large in rural areas
